Liquibook은 쉽습니다. 유치원생도 할 수 있을 만큼 쉽습니다

사용 가능하려면 Liquibook은 통합하기 쉬워야 합니다. 여러분은 다음을 수행해야 합니다:

  • Order Interface를 구현하는 클래스 생성
  • OrderBook 인스턴스화 및 인터페이스 사용
  • OrderBook::perform_callback()을 오버라이딩하여 콜백 처리
  • 호출 스레드에서 OrderBook::perform_callbacks()를 호출하거나 백그라운드 스레드에 신호를 보내 수행

어떻게 하나요...?

  • 내 Order 클래스 사용 - 주문 포인터 타입을 OrderBook 템플릿에 바인딩하기만 하면 됩니다.
    • class MyOrderBook : public OrderBook<MyOrder*> { ... }
  • 공유 포인터 사용 - 공유 포인터 타입을 OrderBook 템플릿 인자로 전달하기만 하면 됩니다.
    • typedef boost::shared_ptr MyOrderPtr;
class MyOrderBook : public OrderBook<MyOrderPtr> { ... }
  • 집계 깊이 레벨 추적 - 자식 OrderBook에서 Depth를 인스턴스화하고, impl::SimpleOrderBook처럼 콜백 처리에서 조작합니다.
    • depth_.add_order(...);
depth_.fill_order(...);
depth_.close_order(...);
depth_.replace_order(...);
depth_.ignore_fill_qty(...);
  • 5단계 이상의 깊이 레벨 추적 - 원하는 숫자로 Depth를 인스턴스화하기만 하면 됩니다.
    • typedef book::Depth<10> MyDepth
  • BBO만 추적 - 한 레벨로 Depth를 인스턴스화하기만 하면 됩니다.
    • typedef book::Depth<10> MyBBO
